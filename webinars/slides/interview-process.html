Theory

questions/01-theory.md

Theory: Format

45-60 minutes, conversational


No coding or whiteboard - just discussion


Rarely a standalone round.

Theory: Topics

TopicFocus
LLM PracticeTemperature, context windows, memory
RAG SystemsProcess, search types, failure points, scaling
Agents & Tool UseComponents, tool selection, termination, security
Testing & EvaluationMetrics, golden datasets, hallucination detection
MonitoringProduction metrics, drift, model testing
Cost & LatencyToken costs, model tiering, caching
Safety & GuardrailsPrompt injection, PII, policy detection

questions/01-theory.md

Theory: RAG Questions

  • Explain the complete RAG process
  • Text vs vector search - when would you use each?
  • How would you process huge PDF reports?
  • How do you handle hallucinations when no info is found in context?
  • What are common RAG failure points and how do you debug them?
  • How do you scale a RAG system to 10M+ articles?

Theory: Agent Questions

  • What makes an AI system agentic?
  • How do agents decide which tool to use?
  • When is an agent the wrong solution?
  • How do you detect and stop infinite planning loops?
  • How do you handle tool failures, retries, and idempotency?
  • What are the biggest security risks with tool-using agents?

Theory: Evaluation & Cost

  • How do you evaluate a chatbot?
  • How do you build a golden dataset for evaluation?
  • How do you detect and mitigate hallucinations?
  • How do you reduce token costs?
  • What is model tiering? When small model vs large LLM?
  • Your app gets 1M queries/day - how do you optimize cost?

Theory: More Questions

  • Monitoring
  • Safety and guardrails
  • Fine-tuning and training
  • LLM theory (transformers, attention, tokenization)
  • ML fundamentals

questions/01-theory.md

Theory: How to Prepare

Focus on practice over theory


Priority topics:

  • RAG systems - build a working pipeline end-to-end
  • Agents - full architecture: planning, tool use, memory, termination
  • Testing and evaluation - how to build golden datasets

Theory: Common Mistakes

  • Describing how to build but not how to evaluate or monitor
  • Not explaining trade-offs - knowing "what" but not "when"

Coding

questions/02-coding.md

Coding: Two Formats

ImplementationAlgorithm
Duration45-90 min25-70 min
StyleProgressive, multi-levelLeetCode-style
EvaluatesCode quality, extensibilityProblem-solving, DSA

Coding: Implementation Rounds

Single problem with multiple levels that build on each other


  • Build a website crawler
  • Refactor 100-120 lines of convoluted code
  • Build a key-value database (SET/GET/DELETE)
  • Unix cd command with symbolic link resolution
  • In-memory database with SQL-like operations
  • Credits management system with complex rules

Coding: Algorithm Rounds

  • RLE encoding
  • Prime numbers between 0 and 100
  • LRU Cache with O(1) time complexity
  • Reverse a linked list (AI-assisted round)
  • Excel column name from column number

Coding: RLE Encoding

aaabbcca → a3b2c2a1

Coding: How to Prepare

  • Solve 75+ easy/medium LeetCode problems
  • Focus on: lists, sets, hash maps, tries, linked lists
  • Build projects with incremental complexity layers
  • Practice narrating your reasoning out loud

AI tools are increasingly allowed. Interviewers watch how you use them.

Coding: Common Mistakes

  • Not clarifying requirements first
  • Writing rigid code that breaks with new requirements
  • Blindly pasting AI output without understanding it

Project Deep Dive

questions/03-project-deep-dive.md

Project Deep Dive: Format

30-60 minutes. Dialogue, not monologue.


  • Conversational deep dive (more common)
  • Prepared presentation (less common)

Project Deep Dive: Questions

  • Walk me through an AI project you built end-to-end
  • Walk me through your most technically challenging project
  • Tell me about a recent/favorite project and some difficulties
  • Describe a challenging prompt engineering problem you solved
  • Tell me about a project you're most proud of

Project Deep Dive: Follow-up Probes

CategoryExample probes
Business contextWhat problem? Why priority? Your actual role?
DecisionsWhy this approach vs alternatives? Trade-offs?
ProblemsWhat went wrong? How did you debug production issues?
ResultsDid it work? How do you know? Actual eval or vibes?
ReflectionWhat would you do differently? What would you explore next?

AI System Design vs Traditional

AI System DesignML System Design
DataContext engineering, chunkingFeature stores, training data
OutputOpen-ended text, codeScores, classifications
DeterminismNon-deterministicGenerally deterministic
EvaluationLLM-as-judge, human evalPrecision/recall/F1
CostPer-token (continuous)Training compute (periodic)
FailuresHallucination, injectionData drift, skew
IterationFast (change prompt)Slow (retrain model)

AI System Design: How to Prepare

Five-step structure:

  1. Problem framing - users, constraints, guardrails
  2. High-level architecture - components and data paths
  3. Deep dive - RAG, tool use, memory, evaluation, safety
  4. Trade-offs - what breaks, detection, graceful degradation
  5. Conclusion - summarize, risks, next iteration

Four repeatable patterns: RAG, feedback loops, hallucination mitigation, scalability/cost optimization

AI System Design: Common Mistakes

  • Jumping to solution without clarifying requirements
  • Treating LLM as source of truth instead of grounding with retrieval/tools
  • Designing only the happy path - no failure modes or monitoring
  • Over-indexing on tool names ("I'd use LangChain") vs explaining why
  • Ignoring cost and latency - token budgets, caching strategies
  • Ignoring safety - prompt injection, data leakage
  • Choosing agents because exciting, not because problem requires autonomy

questions/05-behavioral.md

Behavioral: Format

30-60 minutes, "tell me about a time you..."

Behavioral: Question Categories

CategoryExample
AI-specificWhat side projects have you built with AI?
ConflictTell me about a specific conflict with another person
LeadershipTime you led an initiative or took ownership
DecisionsTechnical misjudgment that led to a delay
FailureMistake you made and what you learned
MotivationWhy this company? Why change now?

Behavioral: How to Prepare

  • Find the company's values
  • For each value, prepare 2-3 stories
  • Use STAR format (Situation, Task, Action, Result)
  • No public values? Use Amazon's Leadership Principles as starting point

Question Categories

  1. Theory - LLMs, RAG, agents, evaluation, cost/latency
  2. Coding - implementation rounds, algorithm problems
  3. Project Deep Dive - past project walkthroughs
  4. AI System Design - designing AI applications

Part 4

Skills That
Get You Hired

interview/03-get-hired.md

Baseline (Expected of Everyone)

  • RAG: Design/implementation questions
  • Vector DBs: Trade-off discussions (Pinecone vs FAISS vs Qdrant)
  • Tool Calling: Agent system design, function calling
  • Python: Coding rounds, project implementations
  • APIs: Integration questions, deployment considerations

Differentiators

  • Evaluation frameworks
  • Cost & latency awareness
  • Observability
  • System diagrams
  • Production experience
  • AI fluency

What Separates Candidates Who Get Hired

From 50+ AI engineer interviews (Fonzi AI)


  • First 5 minutes decide everything. Lead with impact
  • Talk like a builder, not a researcher. Trade-off thinking wins
  • Cost awareness is a superpower. Before/after cost breakdowns impress
  • Honesty about gaps beats bluffing
  • You don't need to be a unicorn - depth in 1-2 areas is enough

Portfolio Strategy (2026)

  • Single high-quality project > multiple certifications
  • Show optimization: prototype to cost-efficient production
  • Clean GitHub: exhaustive README, demos, documentation
  • Include evaluation: every RAG system should have an eval harness

Preparation Advice

  1. Build 2-3 end-to-end projects: RAG app, agent, something deployed
  2. Practice explaining trade-offs aloud - fluency signals experience
  3. Learn evaluation early: Ragas, DeepEval, LLM-as-judge
  4. Show production readiness: Docker, CI/CD, monitoring
  5. Understand cost/latency: caching, batching, model routing
  6. Practice storytelling, not memorized answers

Part 5

After the Interview

interview/04-after-the-interview.md

Handling Rejections

  • Don't take it personally
  • Do a retrospective: understand what went wrong
  • Often nothing was wrong with you - someone was just better, or budget changed, or priorities shifted
  • If you lack a skill, build a project to improve it

Handling Offers & Negotiation

  • Don't rush into agreeing immediately
  • If you have other interviews, finish them first
  • Tell other companies about your offer to speed up their process
  • Multiple offers = ideal negotiation position

Key Takeaways

  • Median process: 4 steps, 2-6 weeks
  • Theory questions focus on RAG, agents, evaluation - practice over theory
  • Coding: implementation rounds (progressive) + algorithm rounds (LeetCode)
  • Project deep dive: the "why" behind decisions matters most
  • AI system design: emerging distinct category, not traditional SD
  • Evaluations are the differentiator - everyone can build, few can measure
  • Lead with impact, not technology names
